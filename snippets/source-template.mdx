## O que é

Explicação conceitual em 2–4 parágrafos. Responder:

- O que essa fonte descreve sobre a pessoa/empresa analisada?
- Que pergunta de negócio ela responde?
- Como ela se encaixa na decisão de crédito?

Evitar jargão técnico. Evitar citar provedor.

## De onde vem

<Note>
  Sempre neutralizar o provedor. Ver [glossário de fontes](/snippets/fontes).
</Note>

- **Tipo de fonte:** bureau de crédito / cadastro oficial / tribunal / cartório / BACEN / lista pública / etc.
- **Cobertura geográfica:** Brasil (nacional) / estados específicos / global.
- **Natureza do dado:** declarado, oficial, consolidado por terceiros, derivado.
- **Base legal:** quando aplicável (LGPD, Lei do Cadastro Positivo, etc).

## Frequência de atualização

| Componente | Atualização |
| ---------- | ----------- |
| Consulta na fonte | real-time ao rodar a operação |
| Cache interno GYRA+ | TTL de N horas/dias (quando houver) |
| Base de origem | diária / semanal / mensal / em tempo real (descrever) |

Explicar em 1 parágrafo o que significa na prática, ex: "um protesto feito hoje aparece na nossa consulta em até X horas."

## Dados entregues

Tabela completa dos campos retornados nesta seção. **Um campo por linha**, sem aninhar demais.

| Campo | Tipo | Nulável | Descrição | Exemplo |
| ----- | ---- | ------- | --------- | ------- |
| `campoA` | `string` | não | O que representa | `"ATIVA"` |
| `campoB` | `number` | sim | ... | `1234.56` |
| `campoC.subCampo` | `string` | sim | ... | `"..."` |

Para estruturas grandes, quebrar em sub-tabelas por subcampo raiz. Não criar tabelas com mais de ~15 linhas, fica ilegível e o RAG não lê bem.

## Como usar na política

Mostrar como criar uma regra concreta a partir desta fonte:

<Steps>
  <Step title="Escolher o campo">
    Qual campo dessa seção é usado. Ex: `sectionX.field`.
  </Step>
  <Step title="Definir o operador">
    Qual operador faz sentido (igual, contém, maior que, soma nos últimos N dias).
  </Step>
  <Step title="Definir o threshold e o peso">
    Valor de corte e impacto da regra na decisão final.
  </Step>
</Steps>

Exemplo prático em JSON (ou screenshot do toolbox):

```json
{
  "field": "sectionX.field",
  "operator": "...",
  "value": "...",
  "weight": 20
}
```

## Limitações e considerações

- Campos que podem vir nulos e por quê.
- Defasagem esperada.
- Casos em que a fonte pode não retornar dado (ex: empresa recém-aberta, CPF sem histórico).
- Custos extras se houver (ex: seção contratada à parte).
