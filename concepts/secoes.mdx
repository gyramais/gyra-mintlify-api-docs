## O que são seções

Um relatório GYRA+ é estruturado em **seções independentes**, cada uma responsável por um tema de análise. As seções são processadas em paralelo e chegam ao frontend em tempo real via WebSocket conforme as integrações são concluídas.

A disponibilidade de cada seção depende do tipo de relatório (SIMPLES, ESSENCIAL, COMPLETO, COMPLETO+) e da política de crédito utilizada.

<CardGroup cols={2}>
  <Card title="SUMMARY" icon="chart-bar">

### Restrições Financeiras

<CardGroup cols={2}>
  <Card title="PEFIN" icon="triangle-exclamation">
    Pendências financeiras registradas em bureaus de crédito. Inclui valor, quantidade, período e comparativo com faturamento.
  </Card>
  <Card title="REFIN" icon="rotate-left">

  </Card>
  <Card title="BAD_CHECK" icon="money-bill-slash">

  </Card>

  </Card>
</CardGroup>

## Status de uma seção

Cada seção pode estar em um dos seguintes estados:

| Status | Descrição |
|--------|-----------|
| Pendente | Aguardando conclusão das integrações |
| Completa | Todos os dados foram recebidos e normalizados |
| Parcial | Algumas integrações concluídas, outras ainda em andamento |
| Erro | Uma ou mais integrações falharam, dados parciais disponíveis |

<Note>
Mesmo com erro em uma integração, o relatório é gerado com os dados disponíveis. A seção `errors` de cada seção detalha o que falhou.
</Note>
