<Info>
  **Resumo:** a fonte de veículos mapeia **propriedade veicular** (RENAVAM) associada a um CPF ou CNPJ: quantos veículos, qual a frota, valor estimado pela tabela FIPE, restrições (alienação, busca e apreensão, roubo). Entra em análise de patrimônio, colateral e detecção de fraude.
</Info>

## O que é

Em crédito PJ e PF, saber se o tomador tem carro, caminhão, frota é informação patrimonial relevante. Para produtos com garantia (CDC, refinanciamento de veículo), é a base. Para análise patrimonial ampla, é um dos indicadores de capacidade ao lado de imóveis.

Responde:

- O tomador tem veículos registrados em seu nome?
- Quantos? Qual a composição (carros, motos, caminhões)?
- Qual o valor total pela FIPE?
- Há restrição (alienação, roubo, busca e apreensão)?

## De onde vem

- **Tipo de fonte:** base RENAVAM (DENATRAN) via agregadores autorizados + tabela FIPE.
- **Cobertura geográfica:** Brasil.
- **Natureza do dado:** semi-público (dados RENAVAM têm acesso regulado, agregadores operam sob autorização).
- **Base legal:** finalidade de análise de crédito e prevenção à fraude.

## Frequência de atualização

| Componente | Atualização |
| ---------- | ----------- |
| Propriedade RENAVAM | mensal (ciclo do DENATRAN) |
| Tabela FIPE | mensal |
| Restrições (alienação, roubo) | 7 a 30 dias dependendo do órgão emissor |

## Dados entregues

| Campo | Descrição | Exemplo |
| ----- | --------- | ------- |
| `vehicles.count` | Quantidade total | `3` |
| `vehicles.totalFipeValue` | Valor total pela FIPE | `187500.00` |
| `vehicles.items[].plate` | Placa | `"ABC1D23"` |
| `vehicles.items[].type` | Tipo | `"CAR"`, `"MOTORCYCLE"`, `"TRUCK"`, `"UTILITY"` |
| `vehicles.items[].brand`, `model`, `year` | Marca, modelo e ano | `"Honda Civic 2022"` |
| `vehicles.items[].fipeValue` | Valor FIPE do veículo | `98500.00` |
| `vehicles.items[].restrictions[]` | Restrições ativas | `["ALIENATION", "THEFT"]` |
| `vehicles.items[].acquisitionDate` | Data de aquisição | `"2022-07-15"` |

## Como usar na política

<Steps>
  <Step title="Regra de alerta por restrição">
    `vehicles.items.hasRestriction IN ["THEFT", "APPREHENSION"]` : `ALERT` ou `DENIED`.
  </Step>
  <Step title="Variável de fórmula de precificação">
    Patrimônio veicular pode entrar na fórmula de `amountFormula` como componente patrimonial.
  </Step>
  <Step title="Checagem para produtos com garantia">
    Integração com produtos específicos (CDC, refinanciamento) usa `vehicles.items[]` filtrado por `restrictions.length == 0`.
  </Step>
</Steps>

## Limitações e considerações

- **Defasagem RENAVAM**: transferências recentes podem levar algumas semanas para refletir.
- **Valor FIPE é referência**, não cotação real de mercado. Veículo em mau estado vale menos que FIPE; veículo raro pode valer mais.
- **Veículo financiado aparece como propriedade**, com `restrictions: ["ALIENATION"]`. Não é do tomador livre até quitar.
- **Dados de motos** têm cobertura menor que carros em algumas UFs.

## Perguntas frequentes

<AccordionGroup>
  <Accordion title="Veículo alienado serve como garantia nova?">
    Não. Veículo alienado já está dado em garantia em outra operação. Para usar como colateral novo, exige quitação e baixa de alienação.
  </Accordion>
  <Accordion title="Como valido que o veículo declarado pelo cliente é o mesmo da consulta?">
    Cruzando placa e chassi entre o que o cliente informa e `vehicles.items[]` do relatório.
  </Accordion>
  <Accordion title="Posso consultar só a placa sem saber o dono?">
    Não por este relatório. A GYRA+ consulta por CPF ou CNPJ, retornando os veículos associados. Consulta por placa isolada é outra via (agregadores específicos).
  </Accordion>
  <Accordion title="Qual nível de relatório inclui veículos?">
    Disponível a partir do **COMPLETO**.
  </Accordion>
</AccordionGroup>
