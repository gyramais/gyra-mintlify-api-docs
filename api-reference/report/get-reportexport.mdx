### Quando usar

Use este endpoint para **disparar** a exportação de um relatório em PDF ou XLS. A geração é assíncrona: a requisição retorna rapidamente e o arquivo fica disponível quando concluído — normalmente entregue por webhook do tipo `REPORT_EXPORTED`. Use essa variante quando não quiser bloquear a requisição esperando o PDF. Para resposta síncrona, use `GET /report/export-sync/{id}`.

### Autenticação

Bearer JWT no header `Authorization`.

### Parâmetros

| Nome | Local | Tipo | Obrigatório | Descrição |
|---|---|---|---|---|
| `id` | `path` | `string` | `sim` | ID do relatório a exportar. |
| `type` | `query` | `string` | `não` | Tipo de export. Valores aceitos: `REPORT` (completo) ou `REPORT_MINIFIED` (resumido). Padrão: `REPORT`. |
| `format` | `query` | `string` | `não` | Formato do arquivo: `pdf`, `excel` ou `both`. Padrão: `pdf`. |

```python Python
import requests

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

<Tip>
Configure um webhook do tipo `REPORT_EXPORTED` para receber o link do arquivo assim que a exportação for concluída. Os links têm validade de 7 dias.
</Tip>
