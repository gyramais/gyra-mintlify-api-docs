### Quando usar

Use este endpoint para verificar quem pode editar relatórios conforme regras de acesso da política. Você pode usar ao montar permissões na interface, controlar governança de mudanças e evitar edição indevida em etapas críticas de decisão.

### Autenticação

Bearer JWT no header `Authorization`.

<RequestExample>

```bash cURL
curl --location 'https://gyra-core.gyramais.com.br/credit-policy/policy/who-can-edit-report?policyId={policyId}' \
--header 'Authorization: Bearer {seu_token_jwt}'
```

```javascript JavaScript
fetch("https://gyra-core.gyramais.com.br/credit-policy/policy/who-can-edit-report?policyId={policyId}", {
  method: "GET",
  headers: {
    "Authorization": "Bearer {seu_token_jwt}",
  },
})
  .then(res => res.json())
  .then(console.log)
```

```python Python
import requests

url = "https://gyra-core.gyramais.com.br/credit-policy/policy/who-can-edit-report"
headers = {
  "Authorization": "Bearer {seu_token_jwt}"
}
params = {"policyId": "{policyId}"}
response = requests.get(url, headers=headers, params=params)
print(response.json())
```
